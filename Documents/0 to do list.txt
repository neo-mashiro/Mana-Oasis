能不能把trigger改成普通的collider？只要合理的设置vectorToBottomAnchor的值，保证gizmo那条线离ladder有段z方向的距离就好了。比如z方向距离0.5，差不多正好是玩家的capsule半径，这样玩家climb的时候既不会collide到梯子，同时手脚又正好能碰到梯子，动画可以很逼真。为了动画的逼真，梯子的segment设的密集一点，不要一个很长的梯子只有几段，看上去好像玩家在爬空气一样，最好每0.25距离就有个梯子的bar。

学习Unity UI 1.0.0 （UGUI）
Unity UI Package也叫做UGUI，是官方的扩展package，就是目前版本的editor里自带的UI工具。目前主要用这个做UI开发，认真学一下。
UI Toolkit是Unity还在开发中的新系统，目前在preview阶段，还有很多bug和功能不全，暂时观望。
UGUI和UI Toolkit都是Unity官方团队开发的，目前uGUI用户最多，也还在更新release，但官方说UI Toolkit是他们的重心。未来应该是需要migrate到UI Toolkit，但是具体要等多少年UI Toolkit才能dominate，就很难说了，估计起码要两三年。
切到UI界面，或者弹出UI菜单的时候，要启用鼠标cursor。
Cursor.visible = true;
Cursor.lockState = CursorLockMode.Confined;

[CRITICAL CHANGE]
1. 现阶段先凑合用老的Input Manager，因为input都比较简单。等基本框架搭起来以后，切换到新的Input System Package，脚本代码再做相应的调整。
不要浪费时间去学习老的Input Manager体系，马上就被淘汰了，新的Input包很牛逼，功能非常强大，可以各种bind事件，根据场景切换功能什么的，是未来的大趋势。
2. Unity的2020 LTS版本出来以后，migrate到新版本的Unity，届时，Unity会支持C# 8.0的新语法和功能，包括nullable reference类型以及default interface implementations之类的。当Interface可以支持default代码实现的时候，传统的OOP继承就基本上没有用武之地了。

！！！！！切换到2020LTS的时候，用不用新的Input System包再看情况，但是所有脚本都要重新过一遍，把标注了【C# 8.0 feature not yet supported】的注释全部拿掉，替换之前的非switch expression语法版本。
